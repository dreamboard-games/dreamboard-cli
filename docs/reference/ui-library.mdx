`@dreamboard/ui-sdk` provides generic React runtime hooks, surfaces, shell
components, board primitives, cards, resources, dice, and default interaction
renderers. For game-specific typing, import generated wrappers from
`@dreamboard/ui-contract` first.

## Canonical split

```tsx
import {
  WorkspaceGameShell,
  useBoardInteractions,
  useInteractionByKey,
  type InteractionParamsOf,
} from "@dreamboard/ui-contract";

import {
  HexGrid,
  ResourceCounter,
  useGameView,
  useGameplayPhase,
  useIsMyTurn,
  usePlayerInfo,
  usePluginActions,
  usePluginSession,
} from "@dreamboard/ui-sdk";
```

Use generated wrappers for:

- `WorkspaceGameShell`
- phase-qualified interaction keys
- `useInteractionByKey`
- `useBoardInteractions`
- `useHexBoardView`
- `InteractionParamsOf`
- workspace zone keys such as `devHand`

Use `@dreamboard/ui-sdk` for generic runtime hooks, components, themes, and
board renderers.

## Runtime hooks

| Hook | Use |
| --- | --- |
| `useGameView()` | Current controlling seat's projected reducer view. |
| `useGameSelector(selector)` | Select a derived slice from plugin state. |
| `useGameplayPhase()` | Current reducer phase name. |
| `useIsMyTurn()` | Whether the controlling seat can act. |
| `useActivePlayers()` | Current active player ids. Prefer generated wrapper when available. |
| `usePlayerTurnOrder()` | Player order. Prefer generated wrapper when available. |
| `useMe()` | Current controlling player metadata. |
| `usePlayerInfo()` | Map of player ids to display metadata. |
| `usePluginSession()` | Session and controlling-player state. |
| `usePluginActions()` | Host actions such as player switching. |
| `useSeatInbox()` | Surface-bucketed projected interaction descriptors. |

Do not use removed action or prompt hooks such as `useActions`,
`usePrompt`, `useChoicePrompt`, or `useGameplayPrompts`. Submit through
generated `useInteractionByKey`, surface handles, `BoardSurface`, or
`useBoardInteractions`.

## Interaction handles

Generated `useInteractionByKey` returns a typed `InteractionHandle`:

```tsx
const handle = useInteractionByKey("playerTurn.offerTrade");

handle?.setInput("give", { brick: 1 });
handle?.setInput("want", { ore: 1 });
await handle?.submitDraft();
```

Important fields:

| Field | Use |
| --- | --- |
| `descriptor` | Reducer-authored metadata and availability. |
| `available` / `unavailableReason` | Disable or explain controls. |
| `draft` | Raw partial local draft params. |
| `values` | Draft params with authored defaults applied. |
| `setInput` / `clearInput` | Edit local draft params. |
| `validateDraft` | Client-side validation from generated schemas. |
| `validateDraftServer` | Server validation without submit. |
| `submit` | Submit explicit params. |
| `submitDraft` | Validate and submit the current draft. |
| `arm` / `disarm` | Set the active surface interaction. |

Default renderers read descriptor fields such as `label`, `shortLabel`, `icon`,
`description`, `emphasis`, `salience`, `group`, `presentation`,
`dispatchPriority`, `available`, `unavailableReason`, `inputs`, `cost`, and
`currentResources`.

## Shell and surfaces

Use [WorkspaceGameShell](/docs/ui/game-shell) for most authored UIs. Compose
lower-level surfaces only when the layout is custom:

```tsx
import {
  BoardSurface,
  HandSurface,
  InboxSurface,
  PanelSurface,
  BlockerSurface,
  ChromeSurface,
} from "@dreamboard/ui-sdk/components";
```

Surfaces bind projected `InteractionDescriptor`s to `InteractionHandle`s. They
do not own legality; legality comes from reducer-authored actors, prompts,
targets, availability, costs, and validation.

## Board primitives

`HexGrid` and `SquareGrid` accept generated or authored board records and can
consume reducer-aware target layers:

```tsx
const board = useBoardInteractions();

<HexGrid
  board={view.board}
  interactiveVertices={board.targetLayers.vertex()}
  interactiveEdges={board.targetLayers.edge()}
  interactiveSpaces={board.targetLayers.space({
    extraInputs: (spaceId) => ({ stealFromPlayerId: targetBySpace[spaceId] }),
  })}
/>;
```

Use generated `useHexBoardView(boardId, { spaces })` when you need to join
static manifest topology with dynamic per-space view overlays before rendering.

## Cards and hands

Use card and hand components with `HandSurface` or a generated hand-zone
renderer:

```tsx
<WorkspaceGameShell
  surfaces={{
    hand: {
      zones: {
        actionHand: {
          renderCardContent: (card, ctx) => (
            <span data-playable={!ctx.disabled}>{card.name ?? card.id}</span>
          ),
        },
      },
    },
  }}
/>;
```

## Resources, dice, and player chrome

`ResourceCounter`, `CostDisplay`, `DiceRoller`, `PlayerRail`, `PlayerInfo`,
`PhaseIndicator`, and `GameEndDisplay` are presentational. They should render
projected view data or interaction handles; they should not inspect reducer
state directly.

`DiceRoller` is display and affordance chrome. Reducer randomness belongs in
`rngInput.*` or `defineEffect` plus `fx.effect(...)`.

```tsx
<DiceRoller
  values={view.diceValues}
  diceCount={2}
  rollAction={{
    handle: rollDiceHandle,
    rollLabel: "Roll dice",
    resultLabel: "Roll total",
  }}
/>;
```
