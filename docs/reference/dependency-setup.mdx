---
title: "Dependency Setup"
description: "Troubleshoot workspace dependency preparation during dreamboard sync."
---

`dreamboard sync` prepares workspace dependencies for you after `package.json`
changes. Most users should not run `pnpm install` manually.

## Requirements

- Node 20+
- `dreamboard` installed with `npm install -g dreamboard`

## Preferred fix

If `dreamboard sync` says dependency tooling is missing, enable Corepack and run
sync again:

```bash
corepack enable
dreamboard sync
```

## Fallback fix

If Corepack is unavailable on your machine, install `pnpm` globally and run
sync again:

```bash
npm install -g pnpm
dreamboard sync
```

## npm lockfile conflicts

Dreamboard manages dependency preparation with its own lockfile. If your
workspace has a `package-lock.json`, remove it and run sync again:

```bash
rm package-lock.json
dreamboard sync
```

## After a failed sync

1. Read the dependency error shown by `dreamboard sync`.
2. Fix the missing tool or package version problem.
3. Run `dreamboard sync` again.
4. When sync succeeds, continue with `dreamboard compile`.
