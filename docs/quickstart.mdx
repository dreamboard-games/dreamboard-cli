---
title: "Quickstart"
description: "Install Dreamboard CLI, create a game, and iterate on manifest-driven scaffolding."
---

Install the CLI:

```bash
npm install -g dreamboard
```

Log in:

```bash
dreamboard login
```

Create a game:

```bash
dreamboard new my-game --description "A trick-taking card game"
cd my-game
```

Edit `rule.md` and `manifest.json`. Use
[CLI](/docs/reference/cli) for command behavior,
[Rule authoring](/docs/reference/rule-authoring) for the rules document and
[Manifest authoring](/docs/reference/manifest-authoring) for the manifest
schema.

Regenerate scaffolded files from your authored source:

```bash
dreamboard sync
```

Compile the current authored head:

```bash
dreamboard compile
```

Start a local server to play the compiled game

```bash
dreamboard dev
```

Useful follow-up commands:

- `dreamboard pull` reconciles authored changes when the remote has advanced.
- `dreamboard status` compares local and remote state.
- `dreamboard clone <game-slug>` pulls an existing game into a local workspace.

For a concise command reference, see [CLI](/docs/reference/cli).

For a full walkthrough, continue with
[Building your first game](/docs/tutorials/building-your-first-game).
